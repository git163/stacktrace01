# C++ 项目
简介
- 1、当前环境是 mac linux, 代码需要兼容 linux。
- 2、当前代码仓需要支持 git 仓库管理。
- 3、代码注释和文档为中文，日志为英文。
- 5、c++代码使用 c++17
- 6、写代码的时候，如果是通用性的，建议写成一个工具，其他项目也好移植和调用，使用第三方库，适当考虑是否封装成一个类，考虑通用和易用性，同时给出常用/推荐的使用用例。
- 7、适当加一些调试日志，方便定位问题，如果循环内会打印的日志太多，会影响查看。
- 8、每次的plan都需要保存到 md 文档，docs 目录下。
- 9、项目中的目录用英文小写命名
- 10、如果单元测试用例失败，如果是代码本身的问题，测试用例没有问题，要求修复代码问题，而不是更改测试用例进行规避
- 11、尽量避免重复造轮子的情况，代码设计要清晰、简单、模块化
- 12、特别要注意，有可能抛异常的地方要进行 ty catch

## 2. c++ 语言
### 2.1 c++命名
- 文件名、类名和函数名使用 `PascalCase`，普通变量使用匈牙利命名。类成员变量的命名是前缀 m_。
- .h文件使用头文件保护防止重复编译
'''c++
// 举例：myheader.h
#ifndef MYHEADER_H
#define MYHEADER_H
'''
### 2.2 c++ 单元测试
使用 Google Test，文件命名 Test<模块名>.cpp，放于 tests/cpp/ 目录。

### 2.3 c++ 编程规范
编程风格采用现代 c++ 编程风格，优先采用引用/std::unique_ptr, 谨慎使用 std::share_ptr,尽量避免裸指针
